Pipeline Stages Explained

💻

Local Code Changes

Developer makes changes to the application code, tests locally, and prepares for deployment. All code modifications begin here before being pushed to the repository.

📤

GitHub Push

Code changes are pushed to the GitHub repository on the main branch. This trigger event initiates the entire CI/CD pipeline automatically.

⚙️

GitHub Actions Build & Push

Automated workflow runs in GitHub Actions. It builds the Docker image, tags it with the commit hash or version, and pushes to Docker Hub registry.

🐳

Docker Hub Registry

Docker image is stored in Docker Hub registry as a centralized, versioned artifact. Enables pulling the exact image version to EC2 when needed.

🔐

SSH to AWS EC2

Securely connects to the EC2 instance via SSH using stored credentials. Executes commands to stop old containers and prepare for the new deployment.

🚀

Docker Container Run

Fresh Docker container is pulled from Docker Hub and runs on EC2 on port 80. Application is now live with the latest code changes deployed.
